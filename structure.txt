KEY PRINCIPLES:

1. **Components Structure**:
   - common/ → Reusable UI components (buttons, inputs, cards)
   - layout/ → Page structure components (header, footer, sidebar)
   - features/ → Feature-specific components (blog-related, auth-related)

2. **Pages**: Each page is a route component that combines smaller components

3. **Hooks**: Custom React hooks for reusable logic

4. **Context**: Global state management with Context API

5. **Services**: 
   - api/ → All API calls
   - utils/ → Helper functions, validators, constants

6. **Routes**: Routing configuration and protected routes

7. **Each component folder contains**:
   - Component.jsx (the component)
   - Component.module.css (styles)
   - index.js (clean exports)

EXAMPLE COMPONENT STRUCTURE:

Button/
├── Button.jsx
├── Button.module.css
└── index.js

This keeps imports clean:
